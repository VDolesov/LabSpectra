# Деплой на Railway

Приложение читает порт из `PORT` и папку данных из `DATA_DIR`, поэтому на
Railway достаточно собрать `Dockerfile`.

## Шаги

1. Railway → проект → **New**.
2. На экране создания сервиса выберите **GitHub Repository**.
3. Выберите репозиторий `VDolesov/LabSpectra`.
4. В настройках деплоя выберите ветку `production-ready`.
   Railway увидит `Dockerfile` и соберёт образ сам.
5. **Volume** (обязательно, иначе данные пропадут при передеплое):
   Railway → сервис → **Volumes → New Volume**, путь монтирования **`/data`**.
   Приложение хранит всё в `DATA_DIR=/data` (задано в `Dockerfile`).
6. **Переменные окружения** (Variables):
   - `ADMIN_PASSWORD` — пароль администратора. Для публичного запуска переменная обязательна.
   - `PORT` Railway задаёт сам — трогать не нужно.
7. Deploy. Публичный URL появится в разделе **Settings → Networking → Generate Domain**.

## Где создать сервер и визуал

- Сервер: в меню на скриншоте нажмите **GitHub Repository** и подключите `VDolesov/LabSpectra`.
- Визуал/ссылка на интерфейс: после успешного деплоя откройте сервис →
  **Settings → Networking → Generate Domain**. Railway даст HTTPS-ссылку, по ней и будет открываться LabSpectra.

## Важно про безопасность

По текущему решению **данные анализов открыты всем по ссылке** — кто угодно может
смотреть, создавать и редактировать. Паролем закрыты раздел «Управление», бэкап
и пересборка реестра. Если это не нужно — не публикуйте домен или добавьте защиту
на весь вход отдельно.

В production-ветке приложение не запустится с переменной `PORT`, если не задан
`ADMIN_PASSWORD`: это защита от случайного публичного запуска с временным паролем.

## Локальный запуск в Docker

```sh
docker build -t labspectra .
docker run -p 8080:8080 -e PORT=8080 -e ADMIN_PASSWORD=секрет -v labspectra-data:/data labspectra
```
Откройте http://localhost:8080
